# 生成時のガイドライン

各タイプのファイル生成時に適用するガイドライン。

## Agent生成時

1. 専門領域を明確に定義
2. 入出力を具体的に記述
3. 使用ツールを必要最小限に制限
4. 自己完結するワークフローを設計

## Command生成時

1. 短く覚えやすい名前
2. 引数は最小限に
3. 1コマンド1目的の原則
4. エラーハンドリングを含める

## Skill生成時

1. descriptionにトリガー条件を明記
2. Progressive Disclosureを意識
3. 関連リソースは別ファイルに分離
4. 500行以内を目標

## Rule生成時

1. 1行で完結
2. 検証可能な内容
3. 既存ルールと矛盾しないか確認
4. グローバル vs プロジェクト固有を判断
