# Rule テンプレート

Rule生成時に使用するテンプレート。

## settings.json形式

```json
{
  "rules": [
    "{{rule_text}}"
  ]
}
```

## CLAUDE.md形式

```markdown
## ルール

- {{rule_text}}
```

## ルール文の書き方

**良い例:**
- `コミットメッセージは日本語で記述する`
- `console.logをデバッグ以外で使用しない`
- `publicメソッドにはJSDocを付ける`
- `テストカバレッジ80%以上を維持する`

**避けるべき例:**
- `コミットメッセージを適切に書く` （曖昧）
- `良いコードを書く` （具体性なし）

**ルール文の原則:**
1. 具体的で実行可能な指示
2. 肯定形または否定形で明確に
3. 1つのルールで1つの制約
4. 検証可能な内容

## 生成時のガイドライン

1. 1行で完結
2. 検証可能な内容
3. 既存ルールと矛盾しないか確認
4. グローバル vs プロジェクト固有を判断
